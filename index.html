Project Overview

UTTA is a specialized chatbot framework designed to assist in teacher training by providing interactive dialogues that model effective teaching strategies. Leveraging the power of Large Language Models (LLMs), UTTA aims to help educators develop their pedagogical skills through simulated student-teacher interactions.

Key Features:

  • DSPy Integration - Optimized prompting for pedagogical responses
  • Knowledge Base Management - Grounding responses in educational content
  • Automated Evaluation Metrics - Measuring teaching effectiveness
  • Interactive Web Interface - User-friendly dialogue environment
  • Context-Aware Responses - Adaptive to student needs and grade levels

Fine-Tuning Approaches

UTTA demonstrates three different approaches to improving LLM performance for teacher-student dialogue interactions:

DSPy: Prompt Optimization

  • Optimizes prompts instead of changing model weights
  • Works with smaller datasets (10-50 dialogues)
  • Immediate results with simpler setup
  • Uses Chain-of-Thought prompting for pedagogical reasoning

OpenAI: Cloud Fine-Tuning

  • Updates actual model weights through cloud service
  • Requires medium-sized datasets (50-100+ dialogues)
  • Balance of control and convenience
  • Handles multi-turn conversations effectively

HuggingFace: Local Fine-Tuning with LoRA

  • Full local control over model and training process
  • Works best with larger datasets (100-1000+ dialogues)
  • Data privacy - all information stays on your machine
  • Uses LoRA for efficient adaptation of model weights

Getting Started

Prerequisites

  • Python 3.10
  • Conda package manager
  • CUDA-capable GPU (for full functionality)
  • OpenAI API key

Quick Installation

# Clone the repository
git clone https://github.com/UVU-AI-Innovate/UTTA.git
cd UTTA

# Make the script executable
chmod +x run_webapp.sh

# Run the setup script
./run_webapp.sh

Visit the GitHub repository for complete installation instructions and documentation.

Evaluation Framework

UTTA includes comprehensive evaluation across 4 key dimensions:

  • Pedagogical Techniques - Use of evidence-based teaching strategies
  • Follow-up Questions - Prompts to check student understanding
  • Response Quality - Length, complexity, and specificity metrics
  • Adaptability - Adjustment to student's prior statements

Implementation Details

The project is structured into several key components:

  • LLM Interfaces - Connections to various language models
  • Knowledge Base - Educational content management
  • Fine-Tuning - Model optimization for teaching
  • Evaluation - Metrics for assessing responses
  • Web Interface - Interactive dialogue environment
